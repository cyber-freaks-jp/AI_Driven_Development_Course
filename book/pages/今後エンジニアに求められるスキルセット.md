# 今後エンジニアに求められるスキルセット

AI時代において、エンジニアに求められるスキルは大きく変化しています。

本章では、今後エンジニアとして生き残り、活躍するために必要な2つの重要なスキルセットについて解説します。

## スキル1：要件定義能力

### なぜ要件定義が重要なのか

AIがコードを自動生成できる時代になると、**何を作るべきか**を明確にする能力がより重要になります。

AIは「どう作るか」は得意ですが、**「何を作るべきか」「なぜ作るのか」は人間が決める必要があります**。

### 求められる要件定義スキル

#### 1. ビジネス理解力

- ビジネスの課題を正確に理解する
- ステークホルダーの要望を整理する
- 本質的な問題を見極める

**例**：
```
❌ 悪い例：「検索機能が欲しい」
✅ 良い例：「ユーザーが過去の注文履歴から特定の商品を素早く見つけられるようにしたい。
           現状、注文履歴が時系列でしか見られず、3ヶ月前の商品を探すのに
           5分以上かかっているため、検索機能が必要」
```

#### 2. 要件の言語化能力

- 曖昧な要望を具体的な要件に落とし込む
- AIに正確に伝えられる形で要件を記述する
- 優先順位を明確にする

**要件定義の例**：
```markdown
## 機能要件

### 検索機能
- ユーザーは注文履歴から商品名で検索できる
- 検索結果は関連度順に表示される
- 検索は部分一致で動作する
- 検索結果は10件ずつページネーション表示

### 非機能要件
- 検索レスポンスは1秒以内
- 10万件の注文履歴でも安定動作
```

#### 3. 要件の分解能力

- 大きな要件を実装可能な単位に分割する
- 依存関係を整理する
- 段階的な実装計画を立てる

### 要件定義スキルの磨き方

1. **ステークホルダーとの対話を増やす**
   - ビジネスサイドと積極的にコミュニケーション
   - 「なぜ」を繰り返し質問する

2. **要件を文書化する習慣をつける**
   - マークダウンで要件を整理する
   - AIに要件を説明してフィードバックをもらう

3. **過去のプロジェクトを振り返る**
   - うまくいった要件定義を分析する
   - 失敗から学ぶ

## スキル2：AIの生成物をチェックする能力

### なぜチェック能力が重要なのか

AIは優れたコードを生成しますが、完璧ではありません。

- **ハルシネーション**：存在しないAPIやライブラリを使うことがある
- **セキュリティの見落とし**：脆弱性のあるコードを生成することがある
- **非効率な実装**：動くが最適ではないコードを生成することがある

**AIの生成物を正しくチェックできる能力**が、エンジニアの重要な役割になります。

### 求められるチェックスキル

#### 1. コードレビュー能力

**セキュリティチェック**：
```python
# AIが生成したコード
def authenticate(username, password):
    query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"
    return db.execute(query)

# ❌ 問題点を発見できる能力が必要
# → SQLインジェクションの脆弱性がある
```

**正しい実装**：
```python
def authenticate(username, password):
    query = "SELECT * FROM users WHERE username=? AND password=?"
    return db.execute(query, (username, password))
```

#### 2. アーキテクチャの妥当性チェック

AIが生成した設計が、システム全体の方針と合っているかを判断する能力。

**チェック項目**：
- 既存システムとの整合性
- スケーラビリティ
- 保守性
- パフォーマンス

#### 3. 動作確認とテスト

AIが生成したコードが、要件を満たしているかを確認する能力。

**確認項目**：
- 正常系の動作確認
- 異常系の動作確認
- エッジケースのテスト
- パフォーマンステスト

### チェックスキルの磨き方

1. **基礎知識を固める**
   - セキュリティの基本を学ぶ
   - 設計パターンを理解する
   - ベストプラクティスを知る

2. **コードレビューの経験を積む**
   - 他の人のコードをレビューする
   - 自分のコードをレビューしてもらう
   - AIが生成したコードを批判的に見る習慣をつける

3. **失敗から学ぶ**
   - バグの原因を深く分析する
   - なぜ見逃したのかを振り返る

## 2つのスキルの関係

この2つのスキルは相互に関連しています。

```
要件定義能力
    ↓
【何を作るか】を明確にする
    ↓
AIがコードを生成
    ↓
【生成物が正しいか】をチェック
    ↓
チェック能力
```

**要件定義が曖昧**だと、AIは正しいコードを生成できません。
**チェック能力が低い**と、AIの間違いを見逃してしまいます。

両方のスキルをバランスよく磨くことが重要です。

## まとめ

今後エンジニアに求められるスキルセット：

### 1. 要件定義能力
- ビジネス理解力
- 要件の言語化能力
- 要件の分解能力

### 2. AIの生成物をチェックする能力
- コードレビュー能力
- アーキテクチャの妥当性チェック
- 動作確認とテスト

この2つのスキルを磨くことで、AI時代においても価値を提供できるエンジニアになることができます。

**AIはツールであり、それを使いこなすのはあなた自身です。**
