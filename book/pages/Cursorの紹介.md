# Cursorの紹介

## はじめに

Cursorは、AIを統合した次世代のコードエディタです。VS Codeをベースに構築されており、強力なAI機能を備えながら、使い慣れたエディタの操作性を保っています。

## Cursorとは

### 基本概念

Cursorは、Anysphere社が開発したAI駆動のコードエディタで、以下の特徴があります：

- VS Code互換のUI/UX
- GPT-4などの最新LLMを統合
- コンテキストを理解したコード生成
- マルチファイル編集対応
- プロジェクト全体の理解

### なぜCursorが人気なのか

1. **VS Code互換**
   - 学習コストが低い
   - 既存の拡張機能が使える
   - キーバインドもそのまま

2. **強力なAI統合**
   - エディタ内で自然言語で指示
   - リアルタイムのコード補完
   - 複数ファイルの同時編集

3. **開発生産性の向上**
   - コード生成の高速化
   - リファクタリングの効率化
   - バグ修正の支援

## 主な機能

### 1. Cursor Tab（AI補完）

**特徴：**
- GitHub Copilot的なリアルタイム補完
- コンテキストを考慮した提案
- Tab キーで即座に適用

**使用例：**
```python
# コメントを書くと...
# ユーザー認証を行う関数

# Tabを押すと自動で補完
def authenticate_user(username, password):
    # パスワードのハッシュ化と検証
    hashed = hash_password(password)
    user = db.get_user(username)
    return verify_password(user.password, hashed)
```

### 2. Cmd+K（インライン編集）

**特徴：**
- 選択範囲を自然言語で編集
- その場で即座に変更
- 差分を確認して適用

**使用例：**
```
選択: 既存の関数
Cmd+K: "エラーハンドリングを追加して"
→ try-catch が追加される
```

### 3. Cmd+L（AIチャット）

**特徴：**
- エディタ右側にチャットパネル
- コードベース全体を参照可能
- ファイル間の関連を理解

**使用例：**
```
質問: "このプロジェクトの認証フローを説明して"
→ 複数ファイルを参照して回答
```

### 4. Composer（マルチファイル編集）

**特徴：**
- 複数ファイルを同時に編集
- プロジェクト全体の変更
- 一貫性のある修正

**使用例：**
```
指示: "ユーザーモデルにemailフィールドを追加"
→ モデル、マイグレーション、APIエンドポイント、
   フロントエンドフォームを同時に更新
```

### 5. @記法（コンテキスト指定）

**特徴：**
- 特定のファイルやコードを参照
- 正確なコンテキスト提供
- 効率的な質問

**記法：**
- `@ファイル名` - 特定ファイル参照
- `@Codebase` - プロジェクト全体
- `@Docs` - ドキュメント参照
- `@Web` - Web検索
- `@Git` - Git履歴

**使用例：**
```
質問: "@auth.py このファイルのセキュリティ問題を指摘して"
```

## 他のツールとの比較

### VS Code + GitHub Copilot

**Cursor の優位点：**
- より自然な会話形式
- マルチファイル編集
- プロジェクト理解度が高い

**GitHub Copilot の優位点：**
- VS Codeそのまま使える
- 価格が安い（$10/月）
- 既存のワークフローを変えない

### ChatGPT

**Cursor の優位点：**
- エディタ内で完結

- 即座にコード適用

**ChatGPT の優位点：**
- ブラウザで使える
- 幅広い質問に対応
- 学習・調査に便利

### Bolt.new

**Cursor の優位点：**
- ローカル開発環境
- 既存プロジェクトの編集
- 詳細な制御

**Bolt.new の優位点：**
- 超高速プロトタイピング
- 環境構築不要

## 実践的な使い方

### 基本ワークフロー

```
1. プロジェクトを開く
   ↓
2. Cmd+L でAIに相談
   「このプロジェクトの構造を教えて」
   ↓
3. 機能追加の相談
   「ユーザー認証機能を追加したい」
   ↓
4. Composer で実装
   複数ファイルを同時編集
   ↓
5. Cmd+K で微調整
   細かい修正を指示
   ↓
6. Tab補完で仕上げ
   コード補完で完成度を上げる
```

### よくある使い方

**1. 新機能の実装**
```
Composer: "REST APIでユーザー管理機能を実装"
→ コントローラー、モデル、ルーティングを自動生成
```

**2. バグ修正**
```
選択: エラーが出ているコード
Cmd+K: "このエラーを修正して"
```

**3. リファクタリング**
```
選択: 長い関数
Cmd+K: "この関数を小さな関数に分割して"
```

**4. テスト作成**
```
Cmd+L: "@auth.py のユニットテストを作成して"
```

**5. ドキュメント作成**
```
Cmd+K: "この関数にdocstringを追加"
```

## Cursorを使うコツ

### 1. 明確な指示

**悪い例：**
```
"ここを直して"
```

**良い例：**
```
"このログイン関数にJWT認証とエラーハンドリングを追加して"
```

### 2. @記法の活用

```
# 特定のファイルを参照
"@models/user.py を参考にして、同じ構造でPostモデルを作成"

# プロジェクト全体を考慮
"@Codebase 全体のエラーハンドリングを統一して"
```

### 3. 段階的な実装

```
1. まず基本実装
2. エラーハンドリング追加
3. テスト作成
4. リファクタリング
```

### 4. Composer の活用場面

- 新機能の追加（複数ファイル）
- データモデルの変更（全体に影響）
- API エンドポイントの追加
- 大規模なリファクタリング

### 5. Rules for AI の設定

`.cursorrules` ファイルでプロジェクト固有のルールを設定：

```
# プロジェクトのコーディング規約
- TypeScriptを使用
- 関数型プログラミングを優先
- エラーは必ずtry-catchで処理
- すべての関数にJSDocを追加
```

## よくある質問

### Q1: VS Codeの設定は引き継げる？

**A:** はい、設定ファイルをインポートできます。拡張機能もほぼそのまま使えます。

### Q2: オフラインで使える？

**A:** 基本的なエディタ機能は使えますが、AI機能はオンライン必須です。

### Q3: プライベートコードは安全？

**A:** Businessプランではデータ保持なしのオプションがあります。Freeプランは学習に使用される可能性があります。

### Q4: どれくらい生産性が上がる？

**A:** タスクによりますが、コード生成で50-70%、リファクタリングで30-50%の時間短縮が報告されています。
